自由演練 10
簡單

文字敏感詞過濾器

貼文發出前的最後防線。設定你的敏感詞清單，把文章貼進來，工具自動標出所有問題詞彙並統計次數。

PRACTICE GOALS
練習字串搜尋與 HTML 高亮標記的實作
理解如何用正則表達式做全域取代
產出一個公關或社群管理人員可用的文字審核工具
TOOL · 互動工具

輸入敏感詞清單，貼入文章，一鍵掃描高亮

在左側輸入要偵測的關鍵字（每行一個），右側貼入需要審查的文字內容，按下「開始掃描」後，所有符合的詞彙會以半透明暖黃色高亮標示，並在下方統計每個關鍵字出現的次數。

keyword-filter.html
敏感詞清單（每行一個）
共 0 個關鍵字
待審查文字
掃描結果 · 共找到 0 處
✓ 文字中未發現任何敏感詞
💡
掃描不區分大小寫。建議先用常見的廣告違規用語測試，例如「保證」「最低價」「免費領取」，確認工具正常運作後，再換成你自己工作場景的關鍵字清單。
HOW TO REPLICATE · 如何自己做出這個工具

用這段 Prompt，讓 Gemini 幫你做一模一樣的工具

1
打開 Gemini，切換到「Canvas」模式
進入 gemini.google.com，點右上角切換到 Canvas 模式，這樣 AI 生成的 HTML 會直接可以預覽。
2
貼入以下指令
把下方指令完整複製，貼入 Gemini 的輸入框，直接送出。
3
等待生成，預覽並存檔
Gemini 會生成一個完整的 HTML 工具。點「預覽」確認功能正常後，點「下載」或「複製」儲存到你的電腦。
→ 複製這段 Prompt 貼入 Gemini
你是一位擅長製作文字工具的前端工程師。 請製作一個「文字敏感詞過濾器」的單頁 HTML 工具，功能如下： 【功能需求】 - 左側：敏感詞清單輸入框（每行一個關鍵字），顯示總關鍵字數 - 右側：待審查文字輸入框 + 「開始掃描」按鈕 - 掃描後在下方顯示原文，所有符合的關鍵字以螢光標記色 highlight - 統計每個關鍵字出現的次數，以標籤形式顯示在結果下方 - 若無符合項目，顯示綠色通過提示 【設計要求】 - 背景 #f5f3ee，強調色 #b5703a，highlight 用半透明暖黃（rgba(201,150,58,.3)） - 字體 Google Fonts（Shippori Mincho + Noto Sans TC） - 左右分欄佈局，圓角 6px，無陰影 請生成完整的單一 HTML 檔案。
CHALLENGE · 進階挑戰

完成基本練習後，試試這些變化

PRACTICE VARIATIONS
01
加入「嚴重程度分級」：讓使用者為每個關鍵字標記「高風險（紅）/ 中等（黃）/ 低風險（綠）」，highlight 顏色對應不同層級。
02
加入「一鍵替換」功能：點擊某個關鍵字標籤後，提示輸入替代詞，自動把文章中的敏感詞換掉。
03
用 Gemini 幫這個工具加上「產業關鍵字包」——讓使用者可以一鍵載入「金融業禁用詞」「醫療廣告法規詞」等預設清單。
