Data Navigation · 2.1

數據引用技巧：精準定位資訊

在 n8n 中，如何從複雜的數據結構中抓取目標資訊是核心技能。本節將詳細講解相對引用與絕對引用兩種「點名抓取法」。相對引用讓工作流程更具彈性，而絕對引用則確保數據路徑的明確性，不管流程多長，都能精準抓到最開頭的客戶電話。

此外，我們將探索 n8n 表達式（Expressions）的強大功能，讓你能動態地構建數據路徑或進行簡單的數據轉換。無論數據巢狀多深，你都能像一位經驗豐富的探險家，輕鬆在數據叢林中找到寶藏。

相對引用 RELATIVE
根據當前節點位置抓取數據
常用於迭代（Loop）中的項目
寫法簡潔：{{ $json.name }}
絕對引用 ABSOLUTE
從工作流程的根源精準定位數據
適合固定來源的數據，不受迭代影響
寫法：{{ $('節點名').item.json.name }}
🧭
選擇原則：相對引用常用於迭代中的數據，絕對引用則適合固定來源的數據。當你需要在 Loop 內部引用 Loop 開始前的初始資料時，絕對引用不可或缺。

Error Handling · 2.2

Optional Chaining ?.：優雅處理空值

數據來源的不確定性經常導致「空值」問題，進而中斷自動化流程。JavaScript 的 Optional Chaining（?.）語法提供了一種優雅的解決方案——有條件地存取物件屬性，確保即使路徑中某個環節為空，流程也能繼續運行。

生活類比 ANALOGY

一般寫法就像直接開門：門存在就進去，門不存在就撞牆當機。?. 的寫法像是：「如果這扇門存在，就開門進去；如果不存在，就安靜地繼續走，不要撞牆。」一個符號，讓流程從脆弱變得堅韌。

🩹
使用範例：原本寫 {{ $json.address.city }}，若 address 不存在就報錯。改成 {{ $json.address?.city }}，address 不存在時回傳 undefined，流程繼續跑。

Decision Making · 2.3

邏輯判斷與分支：建構智慧流程

自動化不僅是執行指令，更是根據條件做出智慧判斷。本節將引導你學習 n8n 中的邏輯判斷節點——If/Else 與 Switch。你將學會如何設定條件、定義不同的執行路徑，讓自動化流程不再是單一路線，而是能根據數據內容動態調整行為。

透過多重條件的組合與巢狀邏輯的設計，你將能夠建構出複雜且強大的決策系統。例如：這份 PDF 的金額超過 10 萬嗎？超過就進主管審核流程，不超過就直接存檔。

🧠
