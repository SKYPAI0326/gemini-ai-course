SECTION 2.1 · 延伸實作

絕對路徑：精準定位跨節點資料

用 ${'節點名稱'}.item.json 語法，從任意節點直接取資料，不受迭代或節點順序影響。

⏱預估時間：15 分鐘
📋3 個步驟
1
相對引用 vs 絕對引用的差別

在 n8n 中，$json.欄位 讀取的是「上一個節點」的輸出——這叫相對引用。當流程有 Loop 迭代，或你需要跨越多個節點取得某個早期節點的資料時，相對引用會失效，因為 $json 的內容隨節點位置改變。

❌ 相對引用（可能出問題）
{{ $json.原始標題 }}

在 Loop 內的節點，
$json 指的是當前
迭代的 item，
不是最初的資料
✅ 絕對引用（精準可靠）
{{ $('讀取 PDF').item.json.標題 }}

不管在哪個節點，
永遠讀取名為
「讀取 PDF」的節點
輸出的標題欄位
2
絕對引用語法與常見模式

絕對引用語法固定格式：{{ $('節點名稱').item.json.欄位名稱 }}。括號內的節點名稱必須與畫布上的節點完全一致（包括空格和大小寫）。

// 基本語法：讀取指定節點的欄位 {{ $('讀取設定檔').item.json.api_key }} // 讀取巢狀欄位 {{ $('解析 JSON').item.json.結果.廠商名稱 }} // 讀取陣列第一筆 {{ $('AI 輸出').item.json.任務列表[0].標題 }} // 在 Loop 內引用 Loop 外的原始資料 {{ $('Webhook').item.json.用戶ID }} // 搭配 ?? 給預設值 {{ $('表單').item.json.備註 ?? '無備註' }}
💡 節點命名決定引用難易度：如果節點叫「HTTP Request 3」，引用時就得寫 ${'HTTP Request 3'}，容易搞混。建議給節點取有意義的中文名，如「呼叫 Gemini」、「解析發票」。
3
實際應用：在 Loop 內保留原始表單資料

最常見的場景：Loop Over Items 處理多個 PDF 時，每次迭代都要帶入表單傳來的「用戶 ID」或「任務名稱」。由於 $json 在 Loop 內指向當前 PDF 的資料，必須用絕對路徑取表單資料。

// 情境：Loop 內的 HTTP Request 節點需要同時用到： // 1. 當前 PDF 的 AI 辨識結果（相對引用） // 2. 表單傳來的用戶 ID（絕對引用） // 在 HTTP Request 的 Body 欄位中： { "pdf_content": {{ $json.ai_result }}, ← 當前 PDF 的結果 "user_id": {{ $('Webhook').item.json.user_id }}, ← 表單原始資料 "task_name": {{ $('Webhook').item.json.task }} ← 表單原始資料 }
⚠️ 引用不存在的節點名稱：如果節點名稱拼錯，n8n 會在執行時報「The node '...' does not exist」錯誤。建議在 Expression 編輯器用左側的節點清單點選，避免手打錯誤。
完成驗證 CHECKPOINT
在 Loop 內成功引用 Loop 外的節點資料

建立一個簡單測試：Webhook → Set（設定一個 user_id 欄位）→ Loop Over Items（設定 3 筆測試資料）→ Set（在 Loop 內的 Set 節點，同時引用 $json.loop資料 和 ${'第一個Set'}.item.json.user_id）。若兩個欄位都能正確讀取，代表絕對引用成功。
