AND / OR 複合條件設計

在 If / Switch 節點中設定多個條件的組合邏輯——AND 要求全部成立，OR 只需一個成立。學會這個，就能處理現實業務中的複雜規則。

📖參考頁面
⏱閱讀約 8 分鐘
AND 與 OR 的邏輯
AND（且）— 所有條件都要成立
「金額 > 10000」AND「客戶等級 = VIP」AND「已付款 = true」
三個條件同時成立才走 True 分支。只要有一個不符，就走 False。
適用：高風險操作需要多重條件確認（大額 + 特定客戶 + 已確認付款）。
OR（或）— 任一條件成立即可
「類型 = 發票」OR「類型 = 收據」OR「類型 = 合約」
任何一個條件成立就走 True。三種類型的文件都要進入 AI 處理流程。
適用：多個觸發條件對應同一個處理邏輯。
在 If 節點設定 AND/OR

打開 If 節點設定，每個條件右邊有「Add condition」新增更多條件。條件群組右上方有一個 AND / OR 切換按鈕——選 AND 代表組內所有條件都要成立，選 OR 代表任一成立即可。

設定方式效果適合場景
同一群組 + AND所有條件必須同時成立需要多重確認的嚴格篩選
同一群組 + OR任一條件成立即過多種觸發條件共用同一流程
多個群組（AND 串 OR）（A AND B）OR（C AND D）複雜業務規則的組合
常見複合條件範例
// 範例一：篩選需要人工審查的訂單 // 條件（AND）： // 1. 金額 > 50000 // 2. 是新客戶（首次下單） // 同時滿足才需要人工審查 // 範例二：觸發緊急通知（OR） // 條件： // 1. 錯誤率 > 10% OR // 2. 回應時間 > 30秒 OR // 3. 成功數 = 0 // 任何一個觸發就發警報 // 範例三：複雜業務規則（AND + OR） // （VIP客戶 AND 金額>10000）OR（企業帳戶 AND 任意金額） // 用兩個 If 節點串聯實現： // 第一個 If：VIP AND 金額>10000 → True 進入快速通道 // False 出口接第二個 If：企業帳戶 → True 進入快速通道
當 If 節點不夠用時：用 Code 節點寫邏輯

如果業務規則非常複雜（超過 3-4 個條件、有巢狀邏輯、需要計算），建議改用 Code 節點直接寫 JavaScript 判斷，可讀性更好，也更容易維護：

// Code 節點中的複雜條件判斷 const data = $input.item.json; const isVIP = data.客戶等級 === 'VIP'; const isEnterprise = data.帳戶類型 === '企業'; const isLargeAmount = data.金額 > 10000; const isNewCustomer = data.訂單數量 === 1; const hasDiscount = data.折扣碼?.length > 0; // 組合複雜條件 const needsReview = (isLargeAmount && isNewCustomer) || (isVIP && isLargeAmount && !hasDiscount) || (isEnterprise && data.金額 > 50000); return [{ json: { ...data, needsReview, reviewReason: needsReview ? '符合人工審查條件' : null } }];
💡 If 節點 vs Code 節點選擇原則：1-3 個條件用 If 節點（視覺清楚）；4 個以上、有計算、有巢狀 → 改 Code 節點（程式可讀性更好，也更容易 debug）。
✅ 記錄判斷理由：在 Code 節點輸出一個 reason 欄位說明為什麼做了這個判斷。日後 debug 時，你能在執行記錄裡直接看到「為什麼這筆資料走了這條路」，省去大量排查時間。
