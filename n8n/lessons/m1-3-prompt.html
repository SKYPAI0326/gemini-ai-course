讓 AI 穩定輸出 JSON 的 Prompt 範本集

合約、發票、會議記錄、郵件分類——四種常見文件的 Prompt 範本，直接複製貼入 n8n HTTP Request 節點即可使用。

📋4 個可直接使用的 Prompt 範本
📖參考頁面，可收藏備用
讓 AI 穩定輸出 JSON 的五個規則
①明確宣告「以純 JSON 格式回覆」，並加上「不要包含任何說明文字」
②提供完整的格式範例（不只說明欄位名稱，要給出資料型別，如字串/數字/陣列）
③說明每個欄位在找不到資料時應填什麼（如 null 或 ""），避免 AI 亂猜
④把文件內容放在 Prompt 最後，用明確的分隔線（如 ---）隔開指令與內容
⑤若 JSON Parse 仍偶爾出錯，在 Code 節點用 raw.match(/\{[\s\S]*\}/)[0] 先截取 JSON 部分再 parse
範本一：合約關鍵資訊提取
合約解析 Prompt 適用：服務合約、採購合約

提取合約中的廠商、金額、日期、類型等關鍵欄位，輸出標準化 JSON。

請讀取以下合約文字，提取關鍵資訊， 以純 JSON 格式回覆，不要包含任何說明文字或 markdown。 JSON 格式如下： { "廠商名稱": "字串，找不到填 null", "合約金額": 數字（不含貨幣符號），找不到填 0, "貨幣": "TWD 或 USD 等，找不到填 'TWD'", "簽約日期": "YYYY-MM-DD，找不到填 null", "到期日": "YYYY-MM-DD，找不到填 null", "合約類型": "採購合約｜服務合約｜保密協定｜其他", "聯絡人": "字串，找不到填 null" } --- 合約文字如下： {{ $json.contract_text }}
範本二：發票與收據解析
發票解析 Prompt 適用：AI 讀取 PDF 後的文字輸出

從 OCR 辨識後的發票文字中提取品項清單與金額資訊。

請讀取以下發票文字，提取關鍵資訊， 以純 JSON 格式回覆，不要包含說明文字或 markdown。 JSON 格式如下（品項為陣列）： { "發票號碼": "字串", "開立日期": "YYYY-MM-DD", "賣方名稱": "字串", "買方名稱": "字串，找不到填 null", "品項": [ { "名稱": "字串", "數量": 數字, "單價": 數字, "小計": 數字 } ], "稅前金額": 數字, "稅額": 數字, "總金額": 數字 } --- 發票文字如下： {{ $json.invoice_text }}
範本三：會議記錄結構化
會議記錄 Prompt 適用：逐字稿或筆記文字

從會議記錄中提取決議事項、待辦任務與負責人。

請讀取以下會議記錄，提取重要資訊， 以純 JSON 格式回覆，不要包含說明文字或 markdown。 JSON 格式如下： { "會議主題": "字串", "日期": "YYYY-MM-DD，推斷不到填 null", "參與者": ["姓名1", "姓名2"], "決議事項": ["事項1", "事項2"], "待辦任務": [ { "任務": "描述", "負責人": "姓名", "截止日": "YYYY-MM-DD 或 null" } ], "下次會議": "YYYY-MM-DD 或 null" } --- 會議記錄如下： {{ $json.meeting_text }}
範本四：郵件分類與摘要
郵件分類 Prompt 適用：Gmail 自動分類流程

判斷郵件類型、緊急程度，並提取關鍵資訊。

請讀取以下郵件內容，分析並分類， 以純 JSON 格式回覆，不要包含說明文字或 markdown。 JSON 格式如下： { "類型": "詢價｜客訴｜合作邀請｜廣告｜內部通知｜其他", "緊急程度": "高｜中｜低", "需要回覆": true 或 false, "摘要": "50字以內的郵件摘要", "關鍵資訊": "金額、日期、產品名稱等重要細節，找不到填 null", "建議回覆": "50字以內的建議回覆方向，不需回覆填 null" } --- 寄件人：{{ $json.from }} 主旨：{{ $json.subject }} 內容： {{ $json.body }}
✅ 使用方式：把 Prompt 貼入 n8n 的 HTTP Request 節點（呼叫 Gemini API）的 Request Body 中，{{ }} 語法引用的欄位名稱換成你的實際欄位名稱，儲存後執行即可。
