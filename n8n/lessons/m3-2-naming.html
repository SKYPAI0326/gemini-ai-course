統一命名規則設計

好的檔名命名規則讓你幾年後還能快速找到任何檔案。設計時要兼顧「機器可排序」和「人眼可讀」，避免日後歸檔混亂。

📖參考頁面
⏱閱讀約 8 分鐘
標準命名結構

最通用的命名結構是：日期_類型_對象_金額或編號。日期放最前，讓資料夾按名稱排序時自然按時間序排列。

命名範例 · 發票
20260414_發票_台積電_125000.pdf
20260414（日期） _ 發票（類型） _ 台積電（廠商） _ 125000（金額）
命名範例 · 合約
20260301_合約_鴻海精密_服務合約_2年.pdf
20260301（簽約日） _ 合約（類型） _ 鴻海精密（廠商） _ 服務合約_2年（補充）
命名規則設計原則
原則說明範例
日期用 YYYYMMDD 讓資料夾按名稱排序 = 按時間排序，不用依賴建立日期 ✅ 20260414 ❌ 2026/4/14
用底線分隔欄位 底線不是特殊字元，在所有作業系統都安全；不要用空格（路徑引用麻煩） ✅ 台積電_發票 ❌ 台積電 發票
重要欄位放前面 排序時前面的欄位優先排。日期通常最重要，其次是類型 20260414_發票_...
避免特殊符號 / \ : * ? " < > | 在各系統有特殊含義，不能用在檔名 ✅ NT125000 ❌ NT$125,000
金額去掉貨幣符號 $ 符號在 shell 有特殊含義；逗號分隔也會造成解析問題 ✅ 125000 ❌ $125,000
在 n8n Code 節點生成規範檔名
// 從 AI 辨識結果生成規範檔名 const data = $input.item.json; // 日期格式化為 YYYYMMDD const date = (data.簽約日期 || data.開立日期 || new Date().toISOString().slice(0,10)) .replace(/-/g, ''); // "2026-04-14" → "20260414" // 清理廠商名稱（移除特殊符號，限制長度） const vendor = (data.廠商名稱 || data.賣方名稱 || '未知廠商') .replace(/[\/\\:*?"<>|]/g, '') // 移除不合法字元 .replace(/\s+/g, '') // 移除空格 .slice(0, 10); // 限制 10 字元內 // 類型正規化 const typeMap = { '發票': '發票', '收據': '收據', '合約': '合約', '採購單': '採購單' }; const type = typeMap[data.合約類型 || data.文件類型] || '文件'; // 金額（無小數點、無符號） const amount = Math.round(data.總金額 || data.合約金額 || 0); // 取得原始副檔名 const ext = $input.item.json.originalFileName?.match(/\.[^.]+$/)?.[0] || '.pdf'; // 組合最終檔名 const newName = `${date}_${type}_${vendor}_${amount}${ext}`; return [{ json: { ...data, newFileName: newName } }];
⚠️ AI 辨識不到欄位時的備援：AI 不一定每次都能正確辨識所有欄位。加入 || '備用值' 確保每個欄位都有值，避免生成出 20260414_undefined_null_0.pdf 這種無效檔名。
