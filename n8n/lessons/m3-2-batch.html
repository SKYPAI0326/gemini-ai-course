Loop + 批次重命名：一次處理整個資料夾

用 Code 節點讀取資料夾清單，再用 Loop Over Items 讓每個 PDF 依序經過 AI 辨識 → 重命名的完整流程。

⏱預估時間：25 分鐘
📋4 個步驟
1
用 Code 節點讀取資料夾所有 PDF

與 3.2 主流程（Watch Folder 監控單一檔案）不同，批次模式需要手動觸發，一次讀取整個資料夾的所有 PDF 並生成 Items 清單。

// 手動觸發 + Code 節點：讀取整個資料夾 const fs = require('fs'); const path = require('path'); const folderPath = '/Users/你的名稱/Documents/待改名'; const files = fs.readdirSync(folderPath) .filter(f => /\.(pdf|PDF)$/.test(f)) // 只要 PDF .filter(f => !f.startsWith('.')) // 排除隱藏檔 .map(fileName => ({ fileName, filePath: path.join(folderPath, fileName), folder: folderPath })); // 每個檔案成為一個 Item return files.map(f => ({ json: f }));
💡 這個 Code 節點的輸出是一個 Items 陣列，每筆 Item 代表一個 PDF 檔案。後面接 Loop Over Items，就能讓每個 PDF 依序被處理。
2
Loop Over Items 設定

搜尋「Loop Over Items」節點加入，將 Code 節點的輸出連接到 Loop 的輸入。Loop 節點會自動把每個 Item 依序送入後續的處理流程。Loop 節點有兩個輸出：Loop（每次迭代的 Item）和 Done（全部完成後觸發）。

⚠️ Gemini API 速率限制：免費方案 15 RPM（每分鐘 15 次）。批次處理 15 個以上 PDF 時，在 Loop 內加入 Wait 節點（設定 5 秒），讓每次 AI 呼叫間隔 5 秒，避免觸發限制。
3
Loop 內：讀取 → AI 辨識 → 重命名

在 Loop 節點的 Loop 輸出接以下流程（與 3.2 主流程相同，只是現在每個 PDF 都會經過）：

// Loop 內的 Code 節點：讀取 PDF 轉 base64 const fs = require('fs'); const data = $input.item.json; const fileContent = fs.readFileSync(data.filePath); const base64 = fileContent.toString('base64'); return [{ json: { ...data, base64 } }]; // → HTTP Request → Gemini API（AI 辨識） // → Code 節點：解析回應，生成新檔名 // → 最後一個 Code 節點：執行重命名 const fs = require('fs'); const path = require('path'); const data = $input.item.json; const newPath = path.join(data.folder, data.newFileName); fs.renameSync(data.filePath, newPath); return [{ json: { success: true, original: data.fileName, renamed: data.newFileName }}];
4
Done 輸出：彙整結果

Loop 的 Done 輸出會在所有 PDF 都處理完後觸發。可以在這裡加一個 Code 節點統計結果，或直接接通知節點發送「批次完成」的摘要。

// Done 輸出的 Code 節點：生成批次摘要 const allResults = $input.all(); const succeeded = allResults.filter(i => i.json.success).length; const failed = allResults.length - succeeded; const summary = [ `✅ 批次重命名完成`, `總計：${allResults.length} 個檔案`, `成功：${succeeded} 個`, `失敗：${failed} 個`, ``, ...allResults.filter(i => i.json.success) .map(i => `${i.json.original} → ${i.json.renamed}`) ].join('\n'); return [{ json: { summary, succeeded, failed } }];
✅ 把摘要接到 Write File 節點，生成一個 重命名日誌_日期.txt 存在資料夾中，日後稽核時可以追蹤哪些檔案被改過。
完成驗證 CHECKPOINT
5 個 PDF 批次重命名成功

準備 5 個測試 PDF（任意內容），執行批次流程。完成後資料夾內應出現 5 個已重命名的 PDF（格式：日期_類型_廠商_金額.pdf），且原始檔名消失。Done 輸出的摘要顯示「成功 5 個，失敗 0 個」。
