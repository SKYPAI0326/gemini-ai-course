預設值處理與資料驗證模式

用 ?? 提供安全預設值，加上常見的資料驗證模式，讓流程對各種異常輸入都有防護。

📖參考頁面
⏱閱讀約 8 分鐘
?? 與 || 的差別

兩個語法都能提供預設值，但觸發條件不同：

語法觸發預設值的條件注意點
值 ?? 預設 值為 null 或 undefined 時才用預設值 空字串 ""、數字 0、false 都不會觸發預設值
值 || 預設 值為任何 falsy 時用預設值 空字串、0、false 都會被當作「沒有值」，可能覆蓋掉合法的 0 或 false
// 情境：表單的「折扣率」欄位，合法值包括 0（不打折） const discount = $input.item.json.折扣率; // 用 || 的問題：0 也會被當成「沒填」 discount || 10 ← 折扣率 0 會變成 10，錯！ // 用 ?? 正確：只有 null/undefined 才用預設值 discount ?? 10 ← 折扣率 0 保留 0，正確
常見預設值模式速查
場景Expression 寫法說明
字串欄位未填{{ $json.備註 ?? '無' }}顯示「無」而非空白
數字欄位為 null{{ $json.金額 ?? 0 }}null 轉 0，避免計算時 NaN
陣列欄位不存在{{ $json.項目 ?? [] }}避免後續 .map() 或 .length 報錯
巢狀欄位 + 預設{{ $json.地址?.縣市 ?? '台灣' }}?. 搭配 ?? 雙重保護
AI 回傳文字{{ $json.candidates?.[0]?.content?.parts?.[0]?.text ?? '' }}Gemini 標準安全讀取
資料驗證：在 Code 節點加入早期錯誤檢查

對於關鍵欄位，與其讓流程在後面某個節點神秘崩潰，不如在第一個 Code 節點就做驗證，找不到就提早回傳錯誤訊息。

// 在流程入口的 Code 節點加入驗證 const data = $input.item.json; // 必填欄位驗證 const required = ['姓名', '電子郵件', '任務類型']; const missing = required.filter(field => !data[field]); if (missing.length > 0) { return [{ json: { success: false, error: `缺少必填欄位：${missing.join(', ')}`, data: null } }]; } // 格式驗證（電子郵件） const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/; if (!emailRegex.test(data.電子郵件)) { return [{ json: { success: false, error: '電子郵件格式錯誤' } }]; } // 驗證通過，繼續流程 return [{ json: { success: true, ...data } }];
💡 驗證後接 If 節點：驗證 Code 節點輸出 success: true/false，後面接 If 節點判斷 $json.success === true，成功的往下走，失敗的走另一條分支（如發送錯誤通知）。
✅ 規則總結：選填欄位一律加 ??；巢狀存取一律加 ?.；關鍵流程入口加驗證 Code 節點。這三個習慣讓你的 Workflow 在各種意外輸入下保持穩健。
