Foundation · 1.1

n8n Desktop 安裝：Make 的本機執行搭檔

你已經學會用 Make 串接雲端服務，但 Make 有兩個根本限制：每月執行次數有上限，而且無法主動讀取你電腦裡的資料夾或檔案。n8n Desktop 就是為了補上這塊缺口而存在——安裝在自己的電腦上，完全免費、無限次執行，並且可以直接操作本機文件系統。

n8n 的節點式介面與 Make 的 Scenario 設計理念相似，已有 Make 基礎的你上手會很快。核心差異在於：n8n 跑在你的電腦裡，任何需要存取本機資料夾、大量迴圈、或不想受次數限制的任務，都交給 n8n 來處理。

💡
Make vs n8n 定位：這兩個工具不是競爭關係，而是分工關係。Make 擅長雲端服務整合與事件觸發；n8n 擅長本機執行、複雜邏輯與大量批次處理。本課程的目標就是讓兩者協同運作。

Connection · 1.2

Cloudflare Tunnel：讓 Make 找到你的 n8n

n8n 裝在你的電腦裡，但 Make、Google Forms 這些雲端服務在網路上看不到你的電腦——它沒有公開的 IP 位址，也沒有對外的網址。Cloudflare Tunnel 解決的正是這個問題：它幫你的本機 n8n 建立一個公開網址，讓雲端服務可以透過這個網址傳送 Webhook 任務進來，整套雲地協作才能成立。

生活類比 ANALOGY

想像 n8n 是你家裡的工作人員，但家裡沒有門牌，外面的快遞（Make、Google Forms）根本不知道要送到哪裡。Cloudflare Tunnel 就是幫你在家門口掛上一個「專屬收件地址」——快遞送到這個地址，任務就會自動轉進你家裡的 n8n，立刻開始處理。

🔗
完全免費，不需技術背景：Cloudflare Tunnel 不需要申請固定 IP、不需要設定路由器，只要跑一個指令就能取得公開網址。對本課程來說，它是讓 Make 與 n8n 溝通的必要橋樑。

Data Essentials · 1.3

JSON 基礎：自動化數據流的骨幹

在自動化流程中，數據以 JSON 格式傳輸是常態。本節將帶你深入理解 JSON 的核心結構，包括物件與陣列的組織方式。掌握 JSON 是駕馭 n8n 的關鍵，它決定了你能否高效地提取、轉換與應用數據，讓自動化工作流程順暢無阻。

你將學習如何從複雜的 JSON 結構中精準提取所需資訊，並結合 Prompting 技巧，確保 AI 輸出的結果能被後續節點正確解讀。例如，要求 AI 回傳 {"vendor": "廠商A", "amount": 50000} 而非自由格式文字。

📚
JSON 是通用語言：幾乎所有現代網路服務（API、Webhook、資料庫）都使用 JSON 格式傳輸數據。掌握它，等於掌握了自動化世界的共同語言。
