SECTION 4.1 · 延伸實作

表單數據接收與解析

Google Forms 提交後，n8n 的 Webhook 節點收到的數據長什麼樣？如何正確解析各欄位、處理多選答案和時間戳記？

⏱預估時間：20 分鐘
📋3 個步驟
1
Webhook 收到的數據結構

Google Forms 透過 Apps Script 傳送到 Make，Make 再轉送到 n8n Webhook 時，n8n 收到的 JSON 結構如下。了解這個結構，才能正確用 Expression 讀取各欄位。

// Make 轉送到 n8n 的 Webhook Body（典型結構） { "timestamp": "2026-04-14T08:30:00.000Z", ← Google Forms 提交時間（UTC） "email": "user@example.com", ← 若開啟收集 Email "task_name": "2026 校慶科技展", ← 表單欄位（自訂鍵名） "task_type": "企劃書", "deadline": "2026-05-01", "notes": "" ← 選填欄位為空字串 }
💡 欄位鍵名由 Apps Script 決定：Apps Script 在轉送資料時，你設定的鍵名（如 task_name）就是 n8n 收到的欄位名稱。在 Apps Script 程式碼中可以完全自訂，不受 Google Forms 的問題文字限制。
2
在 n8n 中讀取與轉換表單數據

Webhook 節點接收到資料後，在後面的 Code 節點做資料整理：轉換時間格式、設定欄位預設值、驗證必填欄位。

// Webhook 後的 Code 節點：整理表單資料 const raw = $input.item.json; // 時間轉換：UTC → 台灣時間（UTC+8） const utcTime = new Date(raw.timestamp); const twTime = new Date(utcTime.getTime() + 8 * 60 * 60 * 1000); const submittedAt = twTime.toISOString().slice(0, 19).replace('T', ' '); // 欄位整理與預設值 const taskName = raw.task_name?.trim() || '未命名任務'; const taskType = raw.task_type || '一般'; const deadline = raw.deadline || null; const notes = raw.notes?.trim() || ''; // 驗證必填欄位 if (!raw.task_name) { return [{ json: { success: false, error: '缺少任務名稱' } }]; } return [{ json: { success: true, submittedAt, taskName, taskType, deadline, notes } }];
3
處理多選答案（核取方塊）

Google Forms 的核取方塊（複選）在 Apps Script 中通常以逗號分隔字串傳出（如 "選項A,選項B,選項C"）。在 n8n 中需要手動拆分成陣列才能逐一處理。

// 多選答案轉換 const selectedOptions = (raw.options || '') .split(',') .map(s => s.trim()) .filter(s => s.length > 0); // 移除空字串 // 範例：options = "PDF改名,日報生成,企劃書" // → selectedOptions = ["PDF改名", "日報生成", "企劃書"] // 判斷是否勾選了特定選項 const needsAI = selectedOptions.includes('AI 生成'); const needsReport = selectedOptions.includes('日報生成');
✅ 在 Apps Script 層就整理格式：如果你能修改 Apps Script，可以在那邊把核取方塊轉成 JSON 陣列再傳送（JSON.stringify(values)），n8n 端就不需要 split 處理。
完成驗證 CHECKPOINT
Webhook 正確接收並解析所有表單欄位

填寫 Google Forms 並提交，在 n8n 的 Webhook 節點執行記錄中確認：
① timestamp 已轉換為台灣時間格式
② 選填欄位空白時有正確預設值
③ 若有多選欄位，已成功拆分成陣列
