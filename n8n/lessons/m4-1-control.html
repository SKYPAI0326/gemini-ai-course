SECTION 4.1 · 延伸實作

遠端指令控制與條件執行

收到表單數據後，如何讓 n8n 根據不同的 task_type 走不同的流程分支？用 Switch 節點實現指令路由，並回傳執行結果給 Make。

⏱預估時間：25 分鐘
📋3 個步驟
1
用 Switch 節點實現任務路由

解析完表單數據後，根據 task_type 欄位決定執行哪條分支。Switch 節點可設定多個條件輸出，每個輸出對應一種任務類型。

OUTPUT 1
企劃書生成
task_type = "企劃書"
→ 呼叫 Gemini 生成企劃內容
OUTPUT 2
PDF 改名
task_type = "PDF改名"
→ 執行批次 PDF 重命名流程
OUTPUT 3
日報生成
task_type = "日報"
→ 觸發當日日報彙整流程
FALLBACK
未知任務
不符合任何條件
→ 回傳錯誤訊息給 Make

在 Switch 節點設定：

// Switch 節點設定 Value 1: {{ $json.taskType }} Routing Rule 1: Equal · "企劃書" → Output 1 Routing Rule 2: Equal · "PDF改名" → Output 2 Routing Rule 3: Equal · "日報" → Output 3 Fallback Output: Output 4（未知任務）
💡 複用現有流程：Switch 的各輸出可以連接到你在 Module 3 建立的流程節點——PDF 批次改名、日報彙整——直接重用，不需要重新建立。
2
執行完成後回傳結果給 Make

n8n 的 Webhook 節點有「Respond to Webhook」功能。在每條分支的末尾加上「Respond to Webhook」節點，將執行結果回傳給等待中的 Make，Make 再根據結果決定後續動作（如發通知）。

// 各分支末端：Respond to Webhook 節點設定 Respond With: JSON Response Body: { "success": {{ $json.success }}, "task_type": {{ $json.taskType }}, "message": {{ $json.message }}, "completed_at": {{ $now.toISO() }} }
⚠️ Make 的等待限制：Make 的 Webhook 等待回應最長 40 秒（免費版）或 60 秒（付費版）。如果任務需要更長時間，在 Webhook 節點先立即回傳 {"status": "accepted"}，完成後再用另一個 HTTP Request 主動 Push 通知到 Make 的第二個 Webhook。
// 長時任務方案：立即接受 + 非同步回報 // Step 1：Webhook 節點立即回傳（讓 Make 不超時） Respond With: JSON · { "status": "accepted", "task_id": {{ $json.taskName }} } Respond When: Immediately // Step 2：任務完成後，HTTP Request 主動 Push Method: POST URL: https://hook.make.com/your-callback-webhook Body: { "task_id": ..., "result": ... }
3
加入執行狀態追蹤（選填）

在每條分支的開始和結束用 Code 節點記錄時間戳，便於事後追蹤哪些遠端指令被執行了、花了多少時間。

// 分支開始的 Code 節點：記錄開始時間 const data = $input.item.json; const startAt = new Date().toISOString(); return [{ json: { ...data, startAt, log: `[${startAt}] 開始執行：${data.taskType}` } }]; // 分支結束的 Code 節點：計算耗時 const data = $input.item.json; const endAt = new Date().toISOString(); const durationMs = new Date(endAt) - new Date(data.startAt); const durationSec = (durationMs / 1000).toFixed(1); return [{ json: { ...data, endAt, duration: `${durationSec} 秒`, success: true, message: `${data.taskType} 執行完成，耗時 ${durationSec} 秒` } }];
✅ Write File 寫執行日誌：將每次遠端指令的執行記錄 Append 到本機的 n8n-log.txt，日後可查閱哪天幾點執行了什麼任務。
完成驗證 CHECKPOINT
遠端指令正確路由並回傳結果

提交 Google Forms 選擇「企劃書」，確認 n8n 走企劃書分支；改選「日報」，確認走日報分支。Make 的執行記錄應顯示收到 n8n 回傳的 {"success": true}，且任務類型正確。
