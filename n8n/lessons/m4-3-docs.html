Google Docs 自動輸出

AI 生成企劃書 JSON 後，如何自動建立 Google Docs、填入格式化內容、並分享給指定成員？用 n8n 的 Google Docs 節點，一鍵完成整份文件。

⏱預估時間：30 分鐘
📋4 個步驟
1
設定 Google Docs 憑證

n8n 內建 Google Docs 節點，使用 OAuth2 連接 Google 帳號。在 n8n 設定憑證後，即可透過節點操作 Google Docs。

設定步驟：

// 設定流程（一次性設定） 1. n8n 介面 → Settings → Credentials → Add Credential 2. 選擇「Google OAuth2 API」 3. 複製 OAuth Redirect URL（n8n 提供的） 4. 到 Google Cloud Console： - 建立專案 → API & Services → Enable APIs - 啟用：Google Docs API、Google Drive API - OAuth consent screen → External → 填入基本資訊 - Credentials → OAuth client ID → Web application - 填入 Authorized redirect URIs（貼入 n8n 的 URL） - 複製 Client ID 和 Client Secret 貼回 n8n 5. 點 Connect → 授權 Google 帳號 → 完成
💡 只需設定一次：憑證設定完成後，後續所有 Google Docs / Google Drive 節點都可以使用同一組 OAuth 憑證，不需要重複設定。
2
建立新 Google Docs 文件

加入「Google Docs」節點，操作選「Create」，自動建立一個新文件並取得文件 ID。

// Google Docs 節點設定（Create） Resource: Document Operation: Create Title: {{ $json.plan.title }}｜企劃書_{{ $now.format('YYYYMMDD') }} // 節點輸出會包含新文件的 documentId // 後面的節點用 {{ $('Create Doc').item.json.documentId }} 引用
💡 Google Docs 節點建立的文件預設放在 Google Drive 根目錄。如需放到指定資料夾，後面再加「Google Drive」節點的 Move 操作。
3
用 Code 節點組合文件內容

Google Docs API 的 batchUpdate 操作需要結構化的 requests 陣列。用 Code 節點把 AI 生成的企劃 JSON 轉換為 Google Docs 支援的格式：

// Code 節點：將企劃 JSON 轉換為 Google Docs 插入請求 const plan = $input.item.json.plan; const docId = $('Create Doc').item.json.documentId; // 組合文件文字（純文字，後面再用 batchUpdate 格式化） const lines = [ plan.title, '', '【活動簡介】', plan.summary, '', '【活動目標】', ...plan.objectives.map((o, i) => `${i + 1}. ${o}`), '', '【執行時程】', ...plan.schedule.map(s => `・${s.phase}（${s.duration}）：${s.tasks}`), '', '【預算估算】', plan.budget_estimate, '', '【預期成效】', plan.expected_outcome, '', '【風險評估】', ...plan.risks.map((r, i) => `${i + 1}. ${r}`), '', '【行銷標語】', plan.marketing_tagline, '', `生成時間：${new Date().toLocaleString('zh-TW', {timeZone: 'Asia/Taipei'})}` ]; const fullText = lines.join('\n'); return [{ json: { docId, fullText, plan, docTitle: plan.title } }];
4
寫入內容並取得分享連結

使用「Google Docs」節點的 Update 操作，將組合好的文字插入文件。接著用「Google Drive」節點取得可分享連結。

// Google Docs 節點（Update） Resource: Document Operation: Update Document ID: {{ $json.docId }} Actions: - Action: Insert Text Text: {{ $json.fullText }} Location: Body Start // 接著加一個 HTTP Request 節點取得文件連結 // Google Drive API 設定分享並取得連結 Method: POST URL: https://www.googleapis.com/drive/v3/files/{{ $json.docId }}/permissions Authentication: Google OAuth2（同一組憑證） Body: { "role": "reader", "type": "anyone" ← 任何人可檢視（不需登入） } // 最後組合可分享的文件 URL // https://docs.google.com/document/d/{docId}/edit
✅ 完成後發送通知：將文件連結加入 Gmail 通知或 Make 的 Line 訊息。收件人點連結就能直接開啟 Google Docs，看到 AI 生成的完整企劃書。
// 最後的 Code 節點：組合通知訊息 const data = $input.item.json; const docUrl = `https://docs.google.com/document/d/${data.docId}/edit`; return [{ json: { ...data, docUrl, notifyMessage: [ `企劃書已自動生成完成！`, ``, `標題：${data.docTitle}`, `連結：${docUrl}`, `生成時間：${new Date().toLocaleString('zh-TW', {timeZone: 'Asia/Taipei'})}` ].join('\n') } }];
完成驗證 CHECKPOINT
Google Docs 企劃書自動生成成功

填寫 Google Forms 輸入活動名稱，提交後約 30-60 秒，Gmail 應收到通知郵件，內含 Google Docs 連結。點開連結確認：① 標題正確 ② 企劃書內容完整（目標、時程、預算等） ③ 文件格式可讀。
