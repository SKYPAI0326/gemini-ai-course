MODULE 03

重型文件工廠：批次處理大師

對付成千上萬的 PDF！學會「迴圈（Looping）」思考術。當別人在手動改名時，你已一鍵處理完一整年的存檔。

學習目標 OUTCOMES
設定資料夾監控，自動觸發文件處理工作流程
實現 PDF 文件的批次自動改名，AI 讀取內容並按規則命名
設定 Schedule Trigger 定時排程，n8n 自主掃描資料夾並以 AI 生成每日摘要日報
Automated Monitoring · 3.1

資料夾監控：Make 做不到的本機觸發

這是 n8n Desktop 相較於 Make 最關鍵的能力差異：Make 只能被動等待雲端訊號（如 Gmail 收信、Google Forms 提交），但它無法主動監控你電腦裡的資料夾。n8n 的「Watch Folder」節點可以——只要你把新檔案丟進指定資料夾，n8n 立刻偵測到並自動啟動處理流程，完全不需人工介入。

透過實作，你將能夠設計出以「本機資料夾」為起點的全自動文件處理系統：發票 PDF 一放進資料夾，AI 自動讀取內容、改名、歸檔，甚至發送通知，整個流程在你的電腦上完成。

📂
Watch Folder 觸發模式：支持即時監聽（文件一落地立刻觸發）或定時掃描（每隔 N 分鐘檢查一次），可依業務需求靈活配置。這個功能在 Make 中沒有對應節點，是 n8n Desktop 的獨有優勢。
01
Watch Folder 節點實作 →
配置 Watch Folder 節點監控資料夾，加入 If 篩選 PDF，建立完整本機觸發流程。
02
監聽策略選擇 →
理解即時監聽與定時掃描的差異，根據任務緊急程度選擇適合的策略。
03
副檔名過濾 →
設定只監控特定副檔名（如 .pdf、.xlsx），避免非目標文件觸發流程。
File Management · 3.2

PDF 批次改名：依規則自動命名

大量 PDF 文件的命名管理常令人頭疼。本節將教你如何利用 n8n 結合 Google AI Studio，從 PDF 內容中提取關鍵資訊，並按照規則實現文件的批次自動改名。

實際案例 EXAMPLE
scan_20260315_001.pdf → 2026_合約_廠商A_50萬.pdf
document_final_v3.pdf → 2026_採購單_供應商B_3.2萬.pdf

無論是根據廠商名稱、合約金額或日期，你都能為文件設定一套智能的命名邏輯，確保所有文件都以統一且具意義的方式命名，易於搜尋與歸檔。

🏷️
AI 辨識 + Prompting：透過 Google AI Studio 讀取 PDF 內容，搭配結構化 Prompt 讓 AI 回傳 JSON 格式的命名資訊，再由 n8n 組合成最終檔名。
01
AI 辨識自動命名實戰 →
Read Binary → Gemini API → Code 解析 → Move File，五步驟完成 PDF 智慧改名。
02
命名規則設計 →
設計統一命名規則（日期_類型_對象_金額），確保一致性與可搜尋性。
03
Loop 批次重命名 →
利用 n8n 的 Loop 與檔案操作節點，將 AI 提取的資訊組合成新檔名並批次執行。
Scheduled Report · 3.3

定時彙整日報：排程掃描 × AI 摘要

這是 n8n 最能展現自主性的場景：完全不依賴任何外部訊號，n8n 自己按時啟動、自己讀取資料夾、自己送給 AI 處理、自己存檔。Make 做不到這件事，因為 Make 必須等待雲端事件才能開始。

本節將用 Schedule Trigger 設定每日排程，配合 Code 節點讀取整個資料夾的文件，由 Gemini AI 生成結構化摘要，最終將 Markdown 日報自動存入指定目錄。設定好後，不需要任何人按下按鈕。

⏱
Schedule Trigger vs Watch Folder：3.1 的 Watch Folder 是「有新檔案才觸發」，而 Schedule Trigger 是「到時間就觸發」，兩者是不同情境的本機觸發策略，各有適用場合。
01
定時 AI 日報實戰 →
Schedule Trigger → Code（讀取資料夾）→ Gemini → Write File，五步驟建立每日自動日報。
02
Cron 語法指南 →
每日、每週、每月不同排程模式的設定方法，以及 Cron 語法的使用時機。
03
多格式日報擴展 →
擴展掃描範圍至 PDF、Excel 等格式，搭配 AI 辨識生成跨格式彙整日報。
