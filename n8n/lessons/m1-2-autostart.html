Cloudflare Tunnel 自動重啟

讓 cloudflared 在 Mac 重開機或登入後自動啟動，不需每次手動執行，讓你的 Webhook 永遠在線。

⏱預估時間：15 分鐘
📋3 個步驟
💡需要使用終端機
1
確認 cloudflared 安裝路徑

在設定自動啟動之前，先確認 cloudflared 執行檔在哪裡。開啟終端機（Terminal），輸入以下指令：

which cloudflared

通常會回傳 /usr/local/bin/cloudflared 或 /opt/homebrew/bin/cloudflared（Apple Silicon Mac）。記下這個路徑，下一步會用到。

💡 若用 .pkg 安裝，路徑通常是 /usr/local/bin/cloudflared；用 Homebrew 安裝的 Apple Silicon Mac 路徑是 /opt/homebrew/bin/cloudflared。
2
建立 launchd 設定檔

macOS 用 launchd 管理背景服務。在終端機中輸入以下指令，建立一個 plist 設定檔（把 /usr/local/bin/cloudflared 換成你的實際路徑）：

# 建立設定檔 cat > ~/Library/LaunchAgents/com.cloudflare.tunnel.plist << 'EOF' <?xml version="1.0" encoding="UTF-8"?> <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd"> <plist version="1.0"> <dict> <key>Label</key> <string>com.cloudflare.tunnel</string> <key>ProgramArguments</key> <array> <string>/usr/local/bin/cloudflared</string> <string>tunnel</string> <string>run</string> </array> <key>RunAtLoad</key> <true/> <key>KeepAlive</key> <true/> </dict> </plist> EOF
⚠️ 前提：需要先完成 1.2 的 Cloudflare Tunnel 設定，並且已用 cloudflared tunnel login 完成登入。自動重啟只是讓已設定好的 Tunnel 開機自動跑，不能跳過初始設定步驟。
3
載入服務並測試

設定檔建立好後，用 launchctl 載入服務讓它立即生效，並驗證是否正常運作：

# 載入服務（現在就啟動） launchctl load ~/Library/LaunchAgents/com.cloudflare.tunnel.plist # 確認服務狀態（看到 PID 代表正在跑） launchctl list | grep cloudflare # 若要停止服務 launchctl unload ~/Library/LaunchAgents/com.cloudflare.tunnel.plist
✅ 重新開機後，cloudflared 會在你登入時自動啟動，Cloudflare Tunnel 的 Public URL 維持不變。你的 Webhook 不再因為重開機而中斷。
完成驗證 CHECKPOINT
重開機後 Webhook 仍能收到請求

重開機後，打開瀏覽器訪問你的 Cloudflare Public URL（如 https://你的名稱.xxx.cfargotunnel.com），若能看到 n8n 的回應（或 Workflow 觸發），代表自動重啟設定成功。
