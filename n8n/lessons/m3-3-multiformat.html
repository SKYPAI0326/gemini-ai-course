SECTION 3.3 · 延伸實作

擴展日報：處理 PDF 與 Excel 格式

3.3 的基礎日報只讀取 .txt/.md 純文字檔。這頁說明如何擴展掃描範圍，讓 AI 也能理解 PDF 和 Excel 的內容並納入彙整。

⏱預估時間：30 分鐘
📋3 個步驟
💡需要完成 3.3 基礎日報
1
讀取不同格式的檔案

純文字（.txt/.md）直接用 fs.readFileSync 讀取即可。PDF 和 Excel 需要額外轉換：PDF 要讀成 base64 傳給 Gemini Vision；Excel（.xlsx）要先轉換成表格文字。

// 修改後的 Code 節點：支援多格式 const fs = require('fs'); const path = require('path'); const folderPath = '/Users/你的名稱/Documents/每日文件'; const today = new Date().toISOString().slice(0, 10); const files = fs.readdirSync(folderPath) .filter(f => /\.(txt|md|pdf|xlsx|xls|csv)$/i.test(f)) .filter(f => !f.startsWith('.')); const fileData = files.map(fileName => { const filePath = path.join(folderPath, fileName); const ext = path.extname(fileName).toLowerCase(); if (['.txt', '.md'].includes(ext)) { // 純文字：直接讀取 const content = fs.readFileSync(filePath, 'utf-8'); return { fileName, type: 'text', content, filePath }; } else if (ext === '.pdf') { // PDF：讀成 base64，給 Gemini Vision 辨識 const base64 = fs.readFileSync(filePath).toString('base64'); return { fileName, type: 'pdf', base64, filePath }; } else if (['.xlsx', '.xls', '.csv'].includes(ext)) { // CSV 可直接讀取；xlsx 需要另外處理（見 Step 2） if (ext === '.csv') { const content = fs.readFileSync(filePath, 'utf-8'); return { fileName, type: 'csv', content, filePath }; } return { fileName, type: 'xlsx', filePath }; } }); return fileData.map(f => ({ json: { ...f, today } }));
2
用 Switch 節點分流不同格式

讀取節點後接 Loop Over Items，每個檔案進入 Loop 後用 Switch 節點根據 $json.type 分流：text/csv 直接送 Gemini；PDF 先轉 base64 再送 Gemini Vision（支援多模態）；xlsx 用 Read Binary Files + 轉換節點先提取文字。

// Switch 節點設定： // 判斷欄位：$json.type // Output 0 條件：Equal to "text" 或 "csv" // Output 1 條件：Equal to "pdf" // Output 2 條件：Equal to "xlsx" // Fallback → 跳過 // PDF 分支：Gemini Vision 的 HTTP Request Body { "contents": [{ "parts": [ { "text": "請閱讀以下 PDF 文件並提取主要內容摘要（200字以內）：" }, { "inline_data": { "mime_type": "application/pdf", "data": "{{ $json.base64 }}" } } ] }] }
⚠️ PDF 大小限制：Gemini API 的 inline_data 方式上限約 20MB。超過的 PDF 需要先分割，或改用 Gemini Files API（先上傳再引用）。工作坊場景下建議測試 PDF 在 5MB 以內。
3
各分支彙整後合併生成日報

所有格式的 AI 摘要完成後，匯入同一個 Code 節點合併：

// Loop 結束後的 Done 輸出：彙整所有摘要 const allItems = $input.all(); const today = allItems[0]?.json?.today || new Date().toISOString().slice(0,10); const sections = allItems.map(item => { const name = item.json.fileName; const summary = item.json.aiSummary || item.json.content?.slice(0, 200); const type = item.json.fileType || '文字'; return `### ${name} (${type})\n${summary}`; }); const report = [ `# 每日工作日報｜${today}`, ``, `> 共彙整 ${allItems.length} 份文件（.txt、.md、.pdf、.xlsx、.csv 混合格式）`, ``, sections.join('\n\n') ].join('\n'); return [{ json: { report, fileName: `${today.replace(/-/g,'')}_日報.md` } }];
✅ 簡化版建議：工作坊時間有限，建議先用純文字（.txt/.md）版本跑通完整流程，再把這頁作為「進階延伸」在課後自行擴展。核心概念（Schedule + AI 摘要 + Write File）完全相同，只是 Step 1 的讀取方式不同。
完成驗證 CHECKPOINT
混合格式日報成功生成

在測試資料夾放入至少 2 種不同格式的文件（如 1 個 .txt + 1 個 .pdf），執行流程。生成的日報 Markdown 檔案應包含兩份文件的摘要，且各自標示了格式類型。
