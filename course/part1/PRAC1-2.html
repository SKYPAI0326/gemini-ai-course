PRACTICE GOALS
學會用四維度框架自我檢查 Prompt 的品質
理解每個維度缺失時會影響哪種回應品質
反覆修改，直到把分數刷到 90 分以上

HOW TO REPLICATE · 如何自己做出這個工具

用這段 Prompt，讓 Gemini 建出你的健診器

1
打開 Gemini Canvas 模式
進入 gemini.google.com，切換 Canvas 模式，讓生成的 HTML 可以即時預覽。
2
複製並貼入以下指令
這段指令使用了「角色 + 任務 + 限制 + 呈現」的完整黃金公式，注意觀察它是如何組合的。
3
測試後存檔
用幾個自己寫過的 Prompt 測試，確認評分邏輯符合你的預期，再下載存檔。
→ 複製這段 Prompt 貼入 Gemini
你是一位擅長製作互動分析工具的前端工程師。 請製作一個「Prompt 品質健診器」的單頁 HTML 工具，功能如下： 【功能需求】 - 一個多行文字輸入框，供使用者貼入他們寫的 Prompt - 一個「健診」按鈕，按下後用 JavaScript 分析輸入文字，依照以下四個維度各給 0～25 分： ① 角色清晰度：偵測是否包含「你是」「扮演」「角色」「身分」等詞彙 ② 任務明確度：偵測是否包含動詞指令（「寫」「整理」「分析」「生成」「製作」） ③ 限制完整度：偵測是否包含「不要」「避免」「限制」「字數」「語氣」等詞彙 ④ 呈現具體度：偵測是否包含「格式」「條列」「表格」「範本」「範例」等詞彙 - 四個評分條（進度條）動態顯示各維度得分 - 顯示總分（0～100）及對應的等級文字（90+: 優秀、70-89: 良好、50-69: 尚可、50以下: 需加強） - 根據低分維度，自動顯示具體的改善建議 【設計要求】 - 背景色 #f5f3ee，卡片背景 #ffffff，邊框 #d8d4cb - 四個維度分別用 #b5703a、#c9963a、#5a7a5a、#6b7fa3 區分顏色 - 字體：標題使用 Shippori Mincho，內文使用 Noto Sans TC（Google Fonts） - 圓角 6px，無陰影，無漸層，整體為無印良品風格 請生成完整的單一 HTML 檔案。
CHALLENGE · 進階挑戰

三個練習，讓你真正掌握 Prompt 品質感

PRACTICE VARIATIONS
01
先隨手寫一個你平常會用的 Prompt，健診後把每個低分維度改善一次，看總分能提升多少。
02
修改 Gemini 指令，讓工具新增「文字長度評估」維度——太短（<20字）或太長（>500字）都要提示。
03
把你最常用的 5 個 Prompt 全部健診一遍，找出你習慣性遺漏的那個維度，針對性練習。
