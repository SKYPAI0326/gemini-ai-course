每次你寫出一個好用的 Prompt，就是在積累一項可以反覆使用的技能資產。把它整理好、存起來，三個月後的你會感謝現在這個你。

Section 01

Prompt 是一種可以積累的技能資產

大多數人使用 AI 的方式是「用完就丟」——每次需要的時候臨時想一個 Prompt，用完關掉，下次再重想一次。這樣永遠停在起點。

真正進階的做法是把每個有效的 Prompt 當成「工具」來對待：存起來、整理好、可以隨時取用。當你的 Prompt 庫累積到一定程度，你會發現自己的工作效率和輸出品質跟同事拉開了明顯差距。

ℹ
Prompt 庫的複利效果：你花 5 分鐘整理一條 Prompt，之後每次使用只需要 30 秒。一年用 50 次，省下超過 4 小時。而且用越多次，Prompt 就越精緻，輸出品質越高。
Section 02

Prompt 公式回顧：四個要素

在第一章我們學過 Prompt 的四要素公式。在整理 Prompt 庫時，這個框架也是你的組織工具——每一條存起來的 Prompt 都應該對照這四個要素來檢查完整度：

①
角色
你是一位…
擅長…的…工程師
②
任務
幫我製作…
輸出…格式
③
限制
不使用外部 library
單一 HTML 檔
④
呈現
RWD 設計
色彩系統指定

存入 Prompt 庫之前，快速掃一遍這四個要素。缺少「限制」的 Prompt 往往產生不穩定的結果；缺少「角色」的 Prompt 輸出會比較平庸。

Section 04

Prompt 範本格式：讓每條指令都可讀可改

存入 Prompt 庫的指令，建議用固定格式記錄——不只存「指令本身」，還要存「使用情境」和「可替換的變數」，讓三個月後的你不需要重新理解就能直接套用：

Prompt 範本格式 可複製的結構
# 名稱：互動式銷售漏斗生成器 # 分類：工具製作類 / 視覺化 # 使用情境：需要向老闆報告業務漏斗時，或要給客戶看轉換率分析 # 變數： 【漏斗層數】← 預設 5 層，可改為 3-7 層 【各層名稱與數字】← 換成你自己的業務數據 【主色調】← 預設鼠尾草綠，可換品牌色 # 指令： 你是一位擅長數據視覺化的資深前端工程師。 請幫我用純 HTML + SVG + JavaScript 製作 一個「互動式銷售漏斗圖」…（完整指令略） # 上次使用：2024-03-15，Q1 業績報告 # 備註：輸出後記得把各層標籤改為中文

這個格式的關鍵在於「變數」欄位——把每次需要替換的地方標記出來，下次使用時只需要填入新數字，不需要重讀整段指令。

✦
存在哪裡？Notion、Obsidian、Apple Notes、或是一個 Excel 試算表都可以。重點是放在你每天都會打開的地方，不然存了也不會用。
Section 05

迭代升級：從 V1 到 V3 的演化

一條好的 Prompt 不是第一次就寫好的。每次使用完，把「哪裡輸出得不夠理想」記下來，下次使用前補進去，Prompt 就會越來越精準。以下是一個真實的迭代過程：

V1
第一版：功能正確，但細節粗糙
幫我做一個銷售漏斗圖，用 HTML
生成了，但沒有互動效果、顏色難看、沒有轉換率標籤
V2
第二版：加入互動和視覺規格
用純 HTML + SVG 製作互動式銷售漏斗，hover 顯示各層人數和轉換率，顏色用鼠尾草綠色系由深到淺
好很多，但 hover tooltip 在手機上位置跑掉，也缺少進場動畫
V3
第三版：加入 RWD 和動畫規格
…RWD，tooltip 不超出螢幕邊界；頁面載入時各層依序淡入；所有資料集中在 JS 陣列方便修改
現在這個版本就是本課程 CH3-3 用的 Prompt，可以直接複製
★
迭代的節奏：每次使用完，花 2 分鐘把「這次哪裡不夠好」加進備註，下次使用前合併進指令。三次迭代之後，大多數 Prompt 就已經非常精準了。
