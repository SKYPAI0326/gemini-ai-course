PART 4 · 成果發表與未來工作流
4.1

工具部署：
存下來、隨時用

你做出來的工具不只是「用一次就關掉的檔案」。把它好好存放、設好捷徑，下次需要的時候兩秒就能打開——而且全部在本機執行，資料不出去，資安零疑慮。

學完這節，你會有
一套整理 AI 工具的資料夾結構，可立即套用
在瀏覽器建立工具捷徑的完整方法
三種部署選項的比較：本機、GitHub Pages、公司內網
Section 01

本機執行：最安全的部署方式

我們整個課程做的工具都是純 HTML 檔案，這代表一件事：它們不需要伺服器就能運作。直接用瀏覽器打開，就是完整功能。

這個特性在企業環境裡是一個巨大的優勢。計算薪資、處理報價、分析客戶數據——這些工作如果上傳到雲端服務，就涉及資料外洩風險。而用本機 HTML 工具，所有運算都在你自己的電腦裡進行，資料從來不會離開你的裝置。

ℹ
純 HTML 工具的本質：它其實就是一個網頁，但不需要連網。瀏覽器（Chrome、Safari、Edge）就是它的「執行環境」，打開即用，關閉即停，沒有安裝、沒有更新、沒有訂閱費。
Section 02

整理你的工具資料夾

工具越做越多之後，最常見的問題是「那個計算機存在哪裡？」建議從一開始就建立一個固定的資料夾結構，讓每個工具都有明確的家。

📁 AI工具庫/
📁 計算類/ 報價計算機.html  ← 客戶報價、快速試算 薪資試算器.html 匯率換算工具.html 📁 排程類/ 排班生成器.html 跨時區會議板.html 📁 視覺化類/ 銷售漏斗圖.html 專案路徑圖.html 📁 其他/ 倒數計時器.html

分類方式不需要完全照上面。重點是讓你自己三秒內找到想要的工具。可以依照工作性質改成「業務類 / 行政類 / 報告類」，或依照使用頻率「每天用 / 每週用 / 偶爾用」來分。

✦
命名技巧：直接用功能命名（「報價計算機」而不是「tool_v3_final」），讓三個月後的你也看得懂。如果有多個版本，在後面加日期：報價計算機_2024-03.html
Section 03

讓工具像 App 一樣好用的三個技巧

工具存好之後，下一步是讓它「夠方便」——打開不超過兩個動作，才會真的被使用。以下三個技巧可以大幅提升工具的可及性：

01
瀏覽器書籤列
把最常用的工具拖拉到瀏覽器的書籤列（Bookmarks Bar）。每次打開瀏覽器都看得到，一秒點開。Chrome / Safari / Edge 都支援，也可以建立「AI 工具」資料夾分類存放。
02
桌面捷徑
Windows：對 HTML 檔案按右鍵 → 傳送到 → 桌面（建立捷徑）。macOS：按住 Option 鍵拖拉檔案到桌面建立別名。雙擊就能用預設瀏覽器打開。
03
釘選到瀏覽器分頁
對正在使用的工具分頁按右鍵 → 釘選分頁（Pin Tab）。它會變成一個小圖示常駐在左側，每次開啟瀏覽器都自動恢復，不佔空間又隨時可用。
Section 04

部署選項比較：本機以外的選擇

如果你想讓團隊成員也能用你做的工具，或需要在不同裝置間存取，可以考慮以下部署方式：

推薦
本機執行
零成本，無需帳號
資料不離開裝置
無需網路連線
適合敏感資料工具
僅限個人裝置使用
選項
GitHub Pages
免費靜態網頁托管
有公開網址可分享
版本控制方便
需要 GitHub 帳號
不適合含敏感資料工具
選項
公司內網
全團隊可存取
資料留在公司內部
可整合到內部入口
需要 IT 部門協助
架設需要一點時間
★
決策原則：工具裡有沒有公司機密或個人資料？有的話就本機執行。只是通用計算工具或展示用圖表？可以考慮 GitHub Pages 讓同事用網址存取，省去每次傳檔的麻煩。
