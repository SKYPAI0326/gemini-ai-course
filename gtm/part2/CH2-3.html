PART 2 · 基礎追蹤與 Chrome 工具運用
2.3

互動追蹤：
精準捕捉按鈕點擊事件

有了上一節取得的 CSS 選擇器，現在我們要教 GTM 認識這個特徵。我們將建立一個專屬的觸發條件，並將其轉化為 GA4 報表中的互動事件。

學完這節，你會
開啟 GTM 的「內建點擊變數」
運用 CSS 選擇器設定「特定元素」的觸發條件
建立並命名規範的 GA4 事件代碼
Section 01

喚醒 點擊變數

GTM 預設處於省電模式，不會去紀錄點擊的細節（如 Class 或 ID）。在開始設定前，我們必須先把這些變數「打開」。

01
進入變數設定
點擊 GTM 左側選單的「變數 (Variables)」，然後點擊右上角的「設定 (Configure)」。
02
勾選點擊相關變數
在右側滑出的清單中找到「點擊 (Clicks)」區塊，將裡面所有的選項（如 Click Element, Click Classes, Click ID）全部打勾，然後關閉清單。
Section 02

建立帶有條件的 Trigger

接下來，我們要告訴 GTM：「只有當點擊的按鈕符合我們上一節找到的特徵時，才算數」。

01
新增觸發條件
點擊「觸發條件」>「新增」。命名為 「點擊 - 特定優惠按鈕」。類型選擇 「點擊 - 所有元素 (All Elements)」。
02
設定過濾條件
將設定改為「部分點擊 (Some Clicks)」。下拉選單選擇 「Click Element」，中間的條件選擇 「符合 CSS 選擇器 (matches CSS selector)」。
03
填入特徵字串
在最後一個欄位，填入上一節找到的值。如果是 ID，填寫 #你的ID；如果是 Class，填寫 .你的Class。我們練習網站的按鈕填寫：.promo-btn-01 a.uagb-buttons-repeater。儲存！
🔎
為何要指定到子層 a？
當你點擊一個按鈕，瀏覽器記錄的「Click Element」是你實際碰到的那層——通常是最內層的 <a>，而不是包在外面的 div。如果選擇器只寫 .promo-btn-01，GTM 比對時找不到這個 class，事件就不會觸發。用 Chrome 的 Inspect 看到哪一層被 highlight，就寫到哪一層。
Section 03

發射 GA4 事件代碼

萬事俱備，只欠東風。我們要把剛剛建立的觸發條件，綁定到一個負責發送資料的代碼上：

  • 新增代碼，類型選擇 「Google Analytics: GA4 事件」。
  • 評估 ID：點欄位旁的選取鈕，選擇在 2.1 建立的 Google 代碼（GTM 會顯示「在這個容器中找到了 Google 代碼」的提示，確認後選取即可）。不需要手動輸入 G- 開頭的 ID。
  • 事件名稱：填寫 click_promo_button。（⚠️請養成使用全小寫英文加底線的好習慣）
  • 觸發條件：選擇剛剛建立好的 「點擊 - 特定優惠按鈕」。儲存。
ℹ
為什麼選取 Google 代碼，而不是手動輸入 ID？
選取方式讓所有事件代碼都引用同一個來源。未來若評估 ID 需要更換，只需修改 CH2-1 建立的那一個 Google 代碼，全部事件代碼自動跟著更新，不必逐一修改。

進階做法：常數變數（Constant Variable）
資深設定者也常在 GTM「變數」中新增一個類型為「常數」的變數，命名為 GA4_ID，內容填入評估 ID。之後所有代碼只要在 ID 欄位選取 {{GA4_ID}} 即可，效果相同。
⚡
按鈕有 href 連結時的注意事項
若按鈕點擊後會跳頁，瀏覽器可能在 GTM 來得及發送事件前就離開了當前頁面。有兩種解法：
① 在 GA4 事件代碼的「更多設定」中，加入欄位 transport_type，值設為 beacon，讓瀏覽器在背景完成發送再跳頁。
② 或把觸發條件類型改為「點擊 - 僅連結 (Just Links)」，GTM 會自動等事件發出後才放行跳頁。
🔍
驗收時間：打開預覽模式，去網站上點擊那個按鈕，看看 Tag Assistant 的時間軸中，這個 GA4 事件代碼是否成功跳到了 Tags Fired 區塊？
