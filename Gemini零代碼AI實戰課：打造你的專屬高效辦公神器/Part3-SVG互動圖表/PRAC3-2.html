實例演練 2

專案時程甘特圖

輸入任務名稱、開始日期、天數，工具自動生成 SVG 甘特圖，幫你視覺化整個專案的時間軸安排，一眼看清哪些任務重疊、哪段是關鍵路徑。

PRACTICE GOALS
理解甘特圖是如何將日期轉換為 SVG 座標的
練習「時間軸」類工具的輸入邏輯與視覺設計
用你現在的實際專案試做一張甘特圖
TOOL · 互動工具

填入任務資訊，自動生成時程甘特圖

輸入各任務的名稱、開始日期（YYYY-MM-DD）和持續天數，點擊「生成甘特圖」後，工具會自動計算時間軸範圍並繪製 SVG 圖表。

甘特圖的核心數學：「任務在時間軸上的 X 位置 = (任務開始日 − 整體最早日) / 總天數 × 圖表寬度」。這個日期到像素的轉換邏輯，就是 SVG 圖表的基本公式，換個場景也完全適用。
HOW TO REPLICATE · 如何自己做出這個工具

讓 Gemini 幫你做一個甘特圖工具

1
先用紙筆畫出你想要的甘特圖樣式
想清楚你的甘特圖需要哪些資訊：只有時間條？還是要有里程碑、進度百分比、負責人？確定後再告訴 Gemini。
2
複製以下指令貼入 Gemini Canvas
注意指令中「自動計算時間軸範圍」的描述——這告訴 Gemini 要根據輸入的日期動態調整比例，不是寫死範圍。
3
用你的真實專案資料測試
把你現在進行中的專案任務輸入進去，確認圖表正確顯示任務重疊情況，再存檔。
→ 複製這段 Prompt 貼入 Gemini
你是一位擅長製作 SVG 數據視覺化工具的前端工程師。 請製作一個「專案時程甘特圖產生器」的單頁 HTML 工具，功能如下： 【功能需求】 - 使用者可輸入多筆任務資料（支援動態新增和刪除），每筆包含： · 任務名稱（文字輸入） · 開始日期（date picker，格式 YYYY-MM-DD） · 持續天數（數字輸入） - 「生成甘特圖」按鈕，按下後用 JavaScript 計算並渲染 SVG 甘特圖 - SVG 甘特圖包含： · 自動計算所有任務的最早開始日與最晚結束日，以此為時間軸範圍 · 水平時間軸，標示日期刻度（每 7 天一個標記） · 每個任務佔一列，左側顯示任務名稱，右側顯示時間條 · 時間條的顏色使用 #5a7a5a 系列（可依任務序號略作深淺變化） · 「今天」的位置用垂直虛線標示 · 滑鼠 hover 時顯示任務的詳細資訊（名稱、日期、天數） 【設計要求】 - 無印良品風格：背景 #f5f3ee，卡片 #ffffff，邊框 #d8d4cb - 甘特條使用 #5a7a5a 主色，透明度漸層處理 - 字體：Shippori Mincho + Noto Sans TC（Google Fonts） - 圓角 6px，無陰影，無漸層（甘特條可有 4px 圓角） 請生成完整的單一 HTML 檔案。
CHALLENGE · 進階挑戰

把甘特圖升級為真正的專案管理工具

PRACTICE VARIATIONS
01
加入「進度百分比」欄位，在甘特條內部顯示完成進度（例如 70% 的甘特條填色，剩下 30% 留空），讓圖表反映實際進度。
02
加入「負責人」欄位，用不同顏色區分不同人負責的任務，方便在同一張圖上看到工作量分配是否均衡。
03
把甘特圖工具整合到你工作中的實際專案，用它向主管或客戶說明時程安排，記錄使用後的回饋。
