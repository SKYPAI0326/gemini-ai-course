PART 3 · 讓數據「動」起來的視覺魔法
3.2

流程視覺化：
實作「動態專案路徑圖」

你花一小時在 PowerPoint 畫的流程圖，老闆一句「這個步驟可以點開看細節嗎」就破功了。動態路徑圖讓每個節點都能 hover 展開說明、點擊切換狀態——讓複雜流程一眼看清楚。

學完這節，你會有
一份「動態專案路徑圖」的完整 Prompt，可直接使用或修改
自訂節點、連線、顏色的追加指令
加入時間軸與進度狀態的進階做法
Section 01

為什麼互動路徑圖比PPT 流程圖更有說服力

PowerPoint 的流程圖有一個根本問題：它是靜態的。你可以把框框畫得很漂亮，但客戶或老闆永遠只能「看」，不能「探索」。而一份複雜的專案流程，光靠看往往不夠。

動態路徑圖解決的，是「讓對方主動參與理解」的問題——他可以 hover 到他有疑問的步驟，展開細節自己讀；可以看到哪個節點是關鍵路徑；可以在你介紹時跟著你的操作走，而不是盯著一張靜止的圖發呆。

✦
實際使用場景：把動態路徑圖的 HTML 打開在瀏覽器，投影到螢幕上做簡報——遠比 PowerPoint 更有互動感，而且不需要任何軟體安裝，瀏覽器就夠了。

以下是一個真實可互動的動態專案路徑圖示範。每個節點都可以 hover 展開詳細說明：

動態專案路徑圖 · 產品上市流程 ↑ 移到節點上看詳細說明
工具解剖 · 動態專案路徑圖 功能拆解
節點 每個步驟一個節點，顏色區分狀態：綠色＝完成、橘色＝進行中、灰色＝待開始
連線 箭頭連接節點，實線＝主路徑，虛線＝分支或平行任務
Hover 滑鼠移到節點時顯示浮動說明卡：任務名稱、負責人、詳細描述、當前狀態
輸入 頁面頂部提供表單，可修改每個節點的名稱、說明、狀態，圖表即時更新
Section 03

完整 Prompt：動態專案路徑圖

動態專案路徑圖 · 完整 Prompt
你是一位擅長數據視覺化的資深前端工程師。 請製作一個「動態專案路徑圖」網頁工具，功能需求如下： 【節點設定】 預設包含 5 個節點，由左至右排列： 1. 需求確認（狀態：完成） 2. 設計規劃（狀態：完成） 3. 開發實作（狀態：進行中） 4. QA 測試（狀態：待開始） 5. 上線發布（狀態：待開始） 每個節點包含以下資料： - 節點名稱（顯示在節點上） - 負責人姓名 - 詳細說明（hover 時顯示） - 預計完成日期 - 狀態（完成 / 進行中 / 待開始 / 延誤） 【視覺呈現】 使用 SVG 繪製，節點以圓角矩形表示，箭頭連接： - 完成：綠色邊框 #2e7d32，綠色打勾圖示 - 進行中：橘色邊框 #e65100，旋轉進度圖示 - 待開始：灰色邊框 #bdbdbd，無圖示 - 延誤：紅色邊框 #c62828，警示圖示 【互動功能】 - 滑鼠移到節點時顯示浮動說明卡（tooltip），內容包含：負責人、詳細說明、預計完成日期 - 點擊節點時，節點高亮（加粗邊框），右側出現詳細資訊面板 - 點擊其他區域時，取消選取 - 頁面載入時，節點依序淡入（間隔 0.15s） 【可編輯性】 頁面頂部有一個「編輯模式」按鈕，點擊後可以修改每個節點的名稱與狀態，確認後更新圖表 【匯出功能】 「下載 SVG」按鈕，讓使用者將圖表下載為 .svg 檔案，可貼入簡報或文件 只使用純 HTML、CSS 與原生 JavaScript 和 SVG， 不引入任何外部套件、D3.js 或 Chart.js。 整體風格簡潔現代，白色背景，適合商業簡報使用。 輸出完整可在瀏覽器直接執行的單一 HTML 檔案，不需要任何說明文字。
Section 04

自訂節點、連線、顏色的追加指令

佈局
「將節點佈局改為由上至下垂直排列，箭頭從上方節點指向下方節點」
適合決策樹或審批流程圖
佈局
「在第二個節點後增加一個分支：分成兩條路徑（A路徑：快速上線；B路徑：完整測試），兩條路徑最後匯合到最後一個節點」
適合需要展示多路徑決策的場景
視覺
「在每個節點下方加上預計完成日期（格式：MM/DD），完成的節點顯示實際完成日期」
讓路徑圖同時承載時間資訊
匯出
「新增一個『複製連結摘要』按鈕，點擊後將所有節點名稱和狀態整理成純文字，格式為：✓ 需求確認（完成） → ⏳ 開發實作（進行中）……，複製到剪貼簿」
讓路徑圖的狀態摘要可以直接貼到 Slack 或 Email
Section 05

進階：加入時間軸與進度狀態

如果你的專案有明確的時間線，可以把路徑圖升級為帶時間軸的版本——每個節點按照日期對齊在橫軸上，讓進度落後或超前一眼可見：

加入時間軸的追加指令
在現有路徑圖下方，新增一條水平時間軸（從今天起算 12 週），功能如下： - 每個節點在時間軸上顯示一個對應的時間標記點 - 標記點的橫向位置對應節點的「預計完成週次」 - 今天的位置在時間軸上用垂直虛線標示為「NOW」 - 已完成的節點：標記點填滿綠色 - 進行中的節點：標記點為橘色空心圓（閃爍動畫） - 若節點的預計完成日已過但狀態仍為「進行中」，標記點改為紅色並顯示「延誤」 時間軸上方的刻度每週一格，顯示週次（W1、W2……）
★
實用建議：路徑圖和時間軸的組合，就是輕量版的甘特圖。不需要買 Project 或 Asana，一個 HTML 檔案就能做到同樣的視覺效果，而且可以客製化任何你需要的功能。
