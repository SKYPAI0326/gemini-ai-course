實例演練 3

會議情境指令模板產生器

輸入會議類型、與會人數、主要目的，自動產生一段針對這場會議量身訂製的完整 AI 指令，讓你的會議記錄整理、摘要生成更精準。

PRACTICE GOALS
學會針對不同情境動態調整 Prompt 的內容
理解「情境變數」如何影響 AI 指令的有效性
產出一批可重複使用的會議 Prompt 模板
TOOL · 互動工具

填入會議情境，自動生成量身指令

不同類型的會議需要不同的 AI 指令策略。客戶提案會、內部週會、跨部門協調會——各自的重點和輸出格式都不一樣。這個工具幫你把情境轉換成精準的指令。

這個工具的核心概念：同樣的黃金公式，因為「情境輸入」不同，就會產生完全不同的指令。學會這個邏輯，你就可以把它應用到任何場景。
HOW TO REPLICATE · 如何自己做出這個工具

讓 Gemini 幫你做一個情境模板產生器

1
想好你最常遇到的工作情境
這個工具的關鍵在於「情境選項」的設計。在告訴 Gemini 之前，先列出你自己工作中最常見的 3～5 種情境。
2
複製並修改以下指令，填入你的情境
把下方指令中的「會議類型」選項替換成你工作上實際會遇到的情境，例如：客服工單、產品發布通知、銷售話術……
3
測試各種情境組合，確認輸出合理後存檔
每一種情境都試一遍，確認生成的 Prompt 真的能用，再下載這個工具。
→ 複製這段 Prompt 貼入 Gemini
你是一位擅長製作互動工具的前端工程師。 請製作一個「情境 Prompt 模板產生器」的單頁 HTML 工具，功能如下： 【功能需求】 - 表單包含以下欄位（下拉選單或文字輸入）： · 情境類型（下拉選單，選項由你設計，至少 5 種） · 任務目的（文字輸入框） · 相關人員（文字輸入框） · 輸出格式（下拉選單：條列式、表格式、段落式） · 語氣要求（下拉選單：正式、簡潔、友善） · 字數限制（下拉選單：300字/500字/不限） - 一個「產生指令模板」按鈕，按下後根據填入的情境組合， 用 JavaScript 動態組合出一段符合黃金公式的完整 AI 指令 - 結果區塊有「複製」按鈕 【指令組合邏輯】 生成的 Prompt 必須包含四個區塊： 【角色】根據情境類型自動填入適合的 AI 身分描述 【任務】結合情境類型和任務目的 【限制】整合語氣要求和字數限制 【呈現格式】對應所選的輸出格式 【設計要求】 - 無印良品風格：背景 #f5f3ee，卡片 #ffffff，邊框 #d8d4cb - 強調色 #b5703a，字體 Shippori Mincho + Noto Sans TC（Google Fonts） - 圓角 6px，無陰影，無漸層 請生成完整的單一 HTML 檔案。
CHALLENGE · 進階挑戰

把這個工具改造成你自己的版本

PRACTICE VARIATIONS
01
把「會議」替換成你工作中最常需要 AI 幫忙的其他情境，例如：「客戶回覆信」、「週報撰寫」、「提案摘要」，做出一個你自己的情境產生器。
02
在工具中加入「歷史記錄」功能——把每次產生的模板儲存在頁面上，方便下次直接取用，不用重新填表。
03
拿這個工具產生的 Prompt 實際去用 Gemini 整理一份真實的會議紀錄，比較有無使用模板的結果差異，記錄在 Prompt 庫裡。
