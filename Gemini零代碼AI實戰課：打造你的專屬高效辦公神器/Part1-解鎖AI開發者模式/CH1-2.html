PART 1 · 解鎖 AI 開發者模式
1.2

溝通心法：
聽懂 AI 的語言——
「角色＋任務＋限制＋呈現」黃金公式

為什麼同樣是「請 AI 做工具」，有人得到一個精美、直接可用的程式，有人卻只得到一堆無法執行的說明文字？差距不在 AI，在於你怎麼問。這一節，我們拆解讓輸出品質倍增的四元素黃金公式。

學完這節，你會
理解為什麼「模糊的指令」會得到「模糊的結果」
掌握黃金公式的四個元素：角色、任務、限制、呈現
看懂好 Prompt 與爛 Prompt 的實際差距，並能自己套用
Section 01

為什麼你之前沒得到想要的結果？

大多數人第一次叫 AI 做工具，輸入的大概是這樣：「幫我做一個計算器」。然後 AI 可能給你一段解釋計算器是什麼的文字，或者給了一個很基本的程式碼，完全不是你腦袋裡想的那個樣子。

這不是 AI 不夠聰明——是因為你給的資訊不夠讓它做出你真正要的東西。「幫我做一個計算器」對 AI 來說，就像你走進一家餐廳說「幫我做一份食物」——廚師沒辦法從這句話裡知道你要吃什麼。

❌ 模糊的指令
「幫我做一個計算器」
AI 自由發揮，可能給你一個最基本的四則運算介面，或者只給你說明文字，完全不符合你的使用場景。
✦ 使用黃金公式
「你是一個專業的前端工程師。幫我製作一個『多層次報價計算機』網頁，讓使用者輸入產品數量與單價，自動計算含稅金額與利潤。只使用純 HTML/CSS/JavaScript，輸出完整可執行的單一 HTML 檔案。」
AI 按照你的規格輸出——欄位正確、邏輯正確、格式正確，直接可用。

同樣的需求，差距來自你提供的資訊量。黃金公式就是幫你確保每次都把四個關鍵資訊告訴 AI。

Section 02

黃金公式的四個元素

黃金公式
角色 ＋ 任務 ＋ 限制 ＋ 呈現
角色 Role 告訴 AI 它是誰
給 AI 一個專業身份，它會用那個身份的標準來思考和輸出。說「你是前端工程師」，它就會寫出工整、符合慣例的程式碼；說「你是 UI 設計師」，它就會更注重視覺排版。
範例
你是一位熟悉商業應用的資深前端工程師
任務 Task 清楚描述要做什麼
說明工具的功能、使用者是誰、要解決什麼問題。越具體越好——「製作一個讓業務人員輸入品項與數量後自動計算含稅報價的工具」遠比「做一個計算器」好。
範例
製作一個多層次報價計算機，使用者可輸入品項、數量、單價，系統自動計算小計、稅金與總金額
限制 Constraint 設定邊界條件
告訴 AI 哪些東西不能做、要避開什麼。這是這門課最重要的一條限制：只用純 HTML/CSS/JS，不引入任何外部套件或框架——這樣輸出的檔案才能在沒網路的環境下也能用。
範例
只使用純 HTML、CSS 與原生 JavaScript，不引入任何外部函式庫或框架
呈現 Format 指定輸出格式
告訴 AI 輸出應該是什麼形式。如果你想要一個可以直接用的檔案，就要明確說「輸出完整的單一 HTML 檔案」；如果不說，AI 可能只給你部分程式碼或摻著說明文字。
範例
輸出完整、可直接在瀏覽器執行的單一 HTML 檔案，不要加任何說明文字
Section 03

實際看看完整 Prompt 長什麼樣

以「會議倒數計時器」為例，以下是用黃金公式寫出來的完整 Prompt。注意每個元素是怎麼自然地串接在一起的：

使用黃金公式的完整 Prompt
// 角色 你是一位熟悉商業應用的資深前端工程師。 // 任務 請製作一個「會議倒數計時器」網頁工具。 功能如下： - 使用者可以輸入會議時長（分鐘） - 點擊「開始」後進入倒數 - 剩餘時間以大字體顯示，格式為「分:秒」 - 時間到時自動出現警告提示 // 限制 只使用純 HTML、CSS 與原生 JavaScript， 不引入任何外部函式庫或框架。 介面風格簡潔，以白色為主色調。 // 呈現 輸出完整、可直接在瀏覽器開啟執行的單一 HTML 檔案， 程式碼之外不需要任何說明文字。
✦
提示：四個元素不需要加標籤或標題，只要內容包含了這四個方向的資訊，AI 就能理解。上面用 // 角色 的格式只是為了讓你看清楚結構，實際使用時不需要寫這些。

現在來看看，如果只是隨手問，會得到什麼——

沒有使用公式的隨手問法
幫我做一個計時器
⚠
這樣的指令，AI 可能給你一個只有數字的基本計時器、可能摻著大量說明文字、可能用了需要安裝的外部套件——而且你得不到任何可以直接存成 .html 使用的東西。
Section 04

辦公室情境的Prompt 範本

以下提供兩個辦公室/行銷情境的黃金公式範本，你可以直接借用或修改後使用：

範本 A · 計算工具類（以薪資試算為例）
你是一位熟悉台灣勞動法規的前端工程師。 請製作一個「薪資試算工具」網頁，功能如下： - 使用者輸入月薪基本薪資 - 自動計算勞健保費用（員工負擔部分） - 顯示實拿金額 - 介面清楚顯示每個項目的金額明細 只使用純 HTML、CSS 與原生 JavaScript， 不引入任何外部套件，介面簡潔專業。 輸出完整可執行的單一 HTML 檔案，不需要附說明。
範本 B · 視覺展示類（以行銷漏斗為例）
你是一位擅長數據視覺化的前端工程師。 請製作一個「互動式行銷漏斗圖」網頁，功能如下： - 顯示曝光、點擊、詢問、成交四個漏斗階段 - 每個階段可以輸入數字，自動計算轉換率 - 滑鼠移到各段時顯示詳細數字 - 視覺風格現代簡潔，適合報告使用 只使用純 HTML、CSS 與原生 JavaScript（可使用 SVG 繪圖）， 不引入任何外部套件。 輸出完整可執行的單一 HTML 檔案，不需要附說明。
★
建立你的 Prompt 庫：每次做出一個好用的工具，把對應的 Prompt 存起來，加到你自己的 Prompt 庫。用過的好指令就是你的資產——這個概念在 Part 4 會詳細說明。
Section 05

調整與迭代：第一次不完美沒關係

黃金公式能大幅提升第一次輸出的品質，但很多時候你還是需要調整。這完全正常——就連有經驗的工程師也常常需要反覆修改才能得到理想結果。

好消息是：在 AI Studio 裡，你可以在同一個對話視窗直接繼續修改，不需要重新寫整個 Prompt：

ℹ
常用的後續調整指令：
「把計算按鈕改成橘色」
「在結果下方增加一個『複製到剪貼簿』的按鈕」
「把字體放大，行距也稍微加寬」
「如果輸入負數，顯示錯誤提示」

每次調整，AI 都會輸出更新後的完整 HTML，你只需要重新複製、貼到原本的檔案並存檔，工具就更新了。不需要自己改任何程式碼。

✦
一個實用技巧：如果調整了幾輪之後，AI 開始出現奇怪的結果或遺忘之前的設定，可以在同一個對話視窗傳這句話：「請根據目前的最新版本，重新輸出完整的 HTML 檔案」——這會讓它重新整合所有修改，重新輸出一個乾淨的版本。
