介面配色方案速查器

選一個主色，選擇風格偏好，工具自動生成五色配色方案，包含背景、卡片、邊框、主色、強調色，每個色票都可以一鍵複製 HEX 碼。

PRACTICE GOALS
理解「輸入一個變數，輸出一套規則」類工具的設計邏輯
練習用 AI 製作帶有色彩計算的視覺工具
拿到一套可直接用在你的 Gemini 工具上的配色方案
TOOL · 互動工具

選主色與風格，一鍵生成配色方案

輸入你喜歡的主色 HEX 碼，或直接點選色盤選色，再選擇你的設計風格，工具會自動生成一套五色配色方案，符合視覺和諧原則。

配色方案的設計邏輯：好的 UI 配色通常只需要「背景 × 卡片 × 邊框 × 主色 × 強調色」五個角色，主色決定個性，其餘四色服務於主色。這個工具示範了「一個輸入 → 一套規則」的典型 UI 工具模型。
HOW TO REPLICATE · 如何自己做出這個工具

讓 Gemini 幫你做一個配色生成工具

1
決定你的色彩系統有幾個角色
在寫指令之前，先想清楚你的工具需要幾種顏色角色，例如：背景/卡片/邊框/主色/強調色，或是更簡化的版本。
2
複製以下指令貼入 Gemini
注意指令中如何描述「色彩計算規則」——這是告訴 AI 用什麼邏輯從主色衍生出其他顏色，而不是讓 AI 隨機決定。
3
測試不同主色，確認生成結果合理
用幾個你工作上常用的品牌色測試，確認生成的配色方案視覺上好看且實用，再存檔。
→ 複製這段 Prompt 貼入 Gemini
你是一位擅長製作互動工具的前端工程師，同時熟悉 UI 配色理論。 請製作一個「介面配色方案速查器」的單頁 HTML 工具，功能如下： 【功能需求】 - 一個顏色選擇器（color picker）+ HEX 輸入框，讓使用者選定主色 - 一個風格下拉選單（無印良品風/商務專業風/清新現代風/深色質感風） - 「生成配色」按鈕，按下後用 JavaScript 計算並生成一套五色配色方案： · 背景色（主色加深度低飽和度版本） · 卡片背景色（比背景稍亮） · 邊框色（介於背景與卡片之間） · 主色（使用者選定的顏色） · 強調色（主色的互補或類比色，依風格調整） - 以色帶（五色並排長條）視覺化顯示配色方案 - 詳細展示每個顏色的：色票預覽、角色標籤（背景/卡片/邊框/主色/強調）、HEX 碼、「複製」按鈕 【色彩計算規則】 使用 HSL 色彩空間進行計算： - 背景色：主色的色相 H，飽和度降至 15–20%，亮度提至 93–95% - 卡片色：比背景亮度再高 2–3%，飽和度再低 3–5% - 邊框色：比背景亮度低 5%，飽和度與背景相同 - 強調色：依風格決定（無印良品：類比色偏暖；商務：互補色降飽和；清新：高亮類比色；深色：主色加深） 【設計要求】 - 工具本身使用無印良品風格：背景 #f5f3ee，卡片 #ffffff，邊框 #d8d4cb - 強調色 #c9963a，字體 Shippori Mincho + Noto Sans TC（Google Fonts） - 圓角 6px，無陰影，無漸層 請生成完整的單一 HTML 檔案。
CHALLENGE · 進階挑戰

把配色工具升級為品牌色規範產生器

PRACTICE VARIATIONS
01
讓工具可以輸入兩個顏色（主色 + 品牌第二色），生成一套完整的雙色系配色方案，適合有固定品牌識別的場景。
02
加入「匯出 CSS 變數」功能——把生成的配色方案輸出為 CSS :root 變數格式，可以直接貼入 Gemini 指令，讓 AI 生成工具時使用你的品牌色。
03
為你目前工作的公司或客戶的品牌主色生成一套配色方案，拿來替換課程中任一個工具的色彩，製作一個「品牌版」的演示工具。
